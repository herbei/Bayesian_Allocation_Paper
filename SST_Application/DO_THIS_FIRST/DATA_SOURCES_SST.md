# UQ26 real-data SST example (observations + computer model output)

This document specifies exactly where the real SST observations and the climate-model SST output come from, and how they are processed to match the UQ26 manuscript setup:

- Regular rectangular lattice (R x C grid)
- Computer output available at all lattice sites (Yc)
- Physical observations available at a subset of lattice sites (Yo)
- Column-major site indexing:  s = r + (c-1)R

The companion scripts in this folder download the raw netCDF files and produce R-ready objects:

- `data_retrieval_sst.R`: downloads OISST observations and builds the observation/grid inputs.
- `download_e3sm_ensembles.R`: downloads the configured E3SM ensemble-member files.
- `data_retrieval_ens.R`: extracts the common grid/month from all downloaded E3SM ensemble files.
- `compute_ocean_laplacian.R`: builds the ocean graph, Laplacian, and optional full eigendecomposition.

---

## 1. Observation-based SST ("real observations")

### Dataset
**NOAA Optimum Interpolation (OI) Sea Surface Temperature (SST) v2, Monthly Mean**

- Data host used here: **NOAA Physical Sciences Laboratory (PSL) THREDDS**
- File: `sst.mnmean.nc`
- Variable: `sst` (monthly mean SST)
- Grid: 1x1 degree lon/lat with centers at lon = 0.5..359.5, lat = 89.5..-89.5 (descending)
- Units: degrees C (`degC`) with `scale_factor = 0.01` and `missing_value = 32767` (per file attributes)

### Download URL
Direct file download (HTTPServer):

- https://psl.noaa.gov/thredds/fileServer/Datasets/noaa.oisst.v2/sst.mnmean.nc

### Notes
- OISST is an observation-based analysis product (blending satellite + in situ sources). The product page at NOAA NCEI describes versioning (v2.1 replacement, coverage, and updates).

---

## 2. Computer model SST ensemble ("computer output")

### Dataset
**CMIP6 E3SM-2-0 historical simulations (monthly ocean, variable `tos`)**

- Data host used here: **ESGF THREDDS node at ORNL**
- Model: E3SM-2-0
- Experiment: `historical`
- Ensemble members downloaded by default: `r1i1p1f1` through `r21i1p1f1`
- Table: `Omon` (monthly ocean)
- Variable: `tos` (Sea Surface Temperature)
- Grid label: `gr` = *regridded to CMIP6 standard 1x1 degree lon/lat grid* (area-average preserving method; stated in file global attributes)
- Time span in the chosen file: **2010-01 through 2014-12** (60 monthly steps)
- Units: degrees C (`degC`) with missing value `1e20`

### Download URL Pattern

Direct file downloads use the ESGF/ORNL THREDDS HTTPServer pattern:

```text
https://esgf-node.ornl.gov/thredds/fileServer/user_pub_work/CMIP6/CMIP/E3SM-Project/E3SM-2-0/historical/<member>/Omon/tos/gr/<version>/tos_Omon_E3SM-2-0_historical_<member>_gr_201001-201412.nc
```


### Notes
- This is **archived model output** from a CMIP6 historical run (i.e., it is not generated by running E3SM locally).
- The file’s global attributes state that the data are distributed under **Creative Commons Attribution 4.0** and refer users to the CMIP6 Terms of Use.

## 3. Ensemble SST files

All configured E3SM-2-0 ensemble members use the same ESGF/ORNL THREDDS layout:

- Activity: `CMIP`
- Institution: `E3SM-Project`
- Model: `E3SM-2-0`
- Experiment: `historical`
- Table: `Omon`
- Variable: `tos`
- Grid label: `gr`
- Default time span: `201001-201412`

The downloader builds URLs of the form:

```text
https://esgf-node.ornl.gov/thredds/fileServer/user_pub_work/CMIP6/CMIP/E3SM-Project/E3SM-2-0/historical/<member>/Omon/tos/gr/<version>/tos_Omon_E3SM-2-0_historical_<member>_gr_201001-201412.nc
```

With `--version=auto`, the script reads the member's THREDDS catalog and tries available version folders from newest to oldest.

## 4. Processing to match the manuscript setup

The script `data_retrieval_sst.R` does the following:

1. **Selects one month** (default `target_ym = "2012-01"`).
2. **Defines a 90 x 159 Pacific-wide lattice** on the shared 1x1-degree grid:
   - lat: 44.5, 43.5, ..., -44.5 (90 points)
   - lon: 130.5, 131.5, ..., 288.5 (159 points)
3. Extracts a single 90x159 SST field from OISST observations: `sst`
4. Vectorizes the 90x159 matrix in **column-major order** to produce:
   - `Xobs_full` (length S): observation-based SST at all grid sites
5. Chooses an observed-site subset `obs_sites` from ocean cells (default N=1000, reproducible random selection with `set.seed(1)`) and defines:
   - `Yo_full = Xobs_full`
   - `Yo = Yo_full[obs_sites]`
6. Saves outputs to `Data/processed/`:
   - `data.rds` and `pacific_sst_data.rds` (observation/grid R objects)
   - `pacific_sst_grid.csv` (grid metadata with r,c,s,lat,lon)
   - `pacific_land_mask.csv` (land/ocean indicator used to sample observation sites)
   - `pacific_obs_sites.csv`, `pacific_Yo_full.csv`, `pacific_Yo.csv`

The script `data_retrieval_ens.R` then reads `Data/raw/tos_Omon_*.nc` and writes:

- `pacific_Yc.csv` (ensemble mean, length S=14310)
- `pacific_Yc_ens_wide.csv`
- `pacific_Yc_ens_long.csv`
- `pacific_Yc_ens_members.csv`
- `pacific_Yc_ensemble_members/pacific_Yc_<member>.csv`

The script `compute_ocean_laplacian.R` then reads `pacific_sst_grid.csv` and `pacific_land_mask.csv` and writes:

- `pacific_ocean_graph_laplacian.rds`
- `pacific_ocean_edges.csv`
- `pacific_ocean_sites_with_local_index.csv`
- `pacific_ocean_laplacian_eigendecomp.rds` when run with `--force-full-eigen`

The resulting objects are designed to plug into the paper’s model:

- `Yo | X, sigma_yo^2 ~ N(H X, sigma_yo^2 I)`
- `Yc | X, beta, sigma_yc^2 ~ N(X + beta, sigma_yc^2 I)`

where `H` is the sampling matrix implied by `obs_sites`.
